# 追補指示書: PR#2 修正 — ブックマークレットを「貼り付け取り込み」方式へ置換

本書は `CODEX_SPEC_phase1-3.md` の §2.2 を**上書き**する追補である。§0の絶対規則はすべて引き続き適用される。
対象は未マージの PR #3(ブランチ `agent/pr2-bookmarklet-intake`)。**同じブランチ上で修正**し、PRタイトルを「PR#2: ページ取り込み(URL/テキスト貼り付け)」に変更すること。

---

## §A. 背景と設計判断(変更してはならない前提)

1. 実機検証の結果、ブックマークレットは最近のChromeのセキュリティ仕様(`javascript:` URLの実行抑止)により、正しく登録しても実行されないことが確認された。**ブックマークレット方式は放棄する**
2. Chrome拡張機能は将来案(フェーズ1.5)としてバックログに残すが、**本PRでは実装しない**
3. 代替は「貼り付け取り込み」方式とする。設計思想は従来と同一: **ユーザー本人が閲覧中のページを、本人の操作で1件ずつ取り込む。一括取得・巡回・自動収集は一切行わない**
4. 既存の `?lead=` URLインテーク(App.jsxのマウント時処理)は**削除せず温存**する。将来のChrome拡張がこの入口を使う。コードに「将来のブラウザ拡張用の入口。削除しないこと」とコメントを付す

## §B. 変更内容

### B1. UI: 取り込みモーダル(LeadTray内)

- 「🔖 ブックマークレットを入手」ボタンとそのモーダル・生成コードを**削除**し、代わりに「**📥 ページから取り込み**」ボタンを設置
- モーダルは2ブロック構成(上下配置。タブでも可):

**ブロック1: URLで取り込み(推奨)**
- input(placeholder: 物件ページのURLを貼り付け)+「取得する」ボタン
- 押下で `POST /api/lead-preview` に `{url}` を送信
- 成功時: 返ってきた `{name, price, rent}` を**編集可能なプレビューフォーム**(物件名/価格万円/家賃円/元URL)に反映 →「トレイに追加」で既存の `addLead()` を呼ぶ(Free 5件上限のメッセージは既存挙動をそのまま利用)
- 失敗時(`ok:false`・タイムアウト・非対応サイト): エラー文言「自動取得できませんでした。下の欄にページの文章を貼り付けてください」を表示し、ブロック2へ視線誘導(スクロールorハイライト)

**ブロック2: ページの文章で取り込み(確実な方法)**
- 案内文: 「物件ページで Ctrl+A → Ctrl+C し、ここに貼り付けてください(スマホは共有→コピー)」
- textarea + 「抽出する」ボタン。**抽出は完全クライアントサイド**:
  - name: 貼り付けテキストの最初の非空行の先頭60文字
  - price: 最初にマッチする `/([0-9,][0-9,.]*)\s*万円/` → 数値(万円)
  - rent: `/(?:賃料|家賃|想定賃料)[^0-9]{0,10}([0-9,]+)\s*円/` → 数値(円)
- 抽出結果は同じ編集可能プレビューフォームに反映 →「トレイに追加」
- 元URL欄は空でよい(ユーザーが手で貼れる任意項目)

### B2. サーバー: `api/lead-preview.js`(新規)

リンクプレビュー(OGP取得)相当の、ユーザー起点・単発のメタデータ取得。以下の制約を**すべて**実装すること:

- メソッド: POSTのみ。入力 `{url}`
- **ドメイン許可リスト**(SSRF対策を兼ねる): hostnameが次のいずれかに一致または `.` 付きで終わる場合のみ許可 — `suumo.jp` / `athome.co.jp` / `homes.co.jp` / `rakumachi.jp` / `kenbiya.com`。それ以外は 400 で `{ok:false, error:"対応サイト外のURLです"}`
- httpsのみ許可。リダイレクトは同一許可ドメイン内で最大2回まで
- タイムアウト6秒、レスポンスは最大500KB・`text/html` のみ読み取り
- User-Agent は正直に名乗る: `RealSimLinkPreview/1.0 (single user-initiated fetch)`
- 抽出(サーバー側): `og:title` → 無ければ `<title>`。価格・家賃はタグ除去した本文先頭20,000字に対しB1と同じ正規表現
- 返却: `{ok:true, name, price, rent}`(取れなかった項目はnull)。取得失敗・パース失敗・ブロック時は `{ok:false}`(500ではなく200で返し、UI側で文章貼り付けへ誘導)
- レート制限: IPあたり20回/日(モジュールスコープのMapによる簡易実装でよい。ベストエフォート)
- 認証は不要(Freeユーザーにも開放する機能)。**AnthropicのAPIは使わない**(単なるHTML取得とregex)

### B3. 文言・免責

モーダル下部に小さく表示: 「取り込みは、あなたが閲覧中のページをあなたの操作で1件ずつ保存する機能です。サイトの一括取得は行いません。取得内容は必ず元ページで確認してください。」

## §C. 受け入れテスト(PR説明に結果を記載)

1. `npm run build` 成功
2. 対応ドメインの実URL(例: suumo.jpの中古マンション詳細)で「取得する」→ 成功時はプレビューに名称・価格が入る/ブロック等で失敗時はブロック2への誘導文言が出る(**どちらの結果でも受け入れ可**。挙動が正しいことが基準)
3. 非対応ドメイン(例: `https://example.com`)で「対応サイト外」エラーが返る
4. SUUMOページの本文をコピペ→「抽出する」で、価格(万円)と家賃または名称が抽出されプレビューに入る
5. 「トレイに追加」でトレイに反映され、Free上限(5件)超過時に既存メッセージが出る
6. `?lead=` インテークが引き続き動作する(回帰確認)
7. ブックマークレット関連のUI・文言が残っていない
