# 追補指示書: 取り込みモーダルのUX修正(state残存・ドラッグ誤閉じ)

対象: `agent/pr2-bookmarklet-intake` ブランチ(PR #3)。**同じブランチ上で追加修正**すること。§0の絶対規則は引き続き適用。

## 症状(2件)

1. 「📥 ページから取り込み」モーダルを2回目に開くと、前回入力したURL/テキストが残っている
2. 残ったURLをテキスト選択で消そうとすると、input/textarea内で始まったドラッグの指を**モーダル背景の上で離した瞬間にモーダルが閉じる**

## 原因

1. モーダルを閉じるときに入力state(url, text, プレビュー結果, エラー文言)がリセットされていない
2. 背景オーバーレイの閉じる判定が `onClick` になっており、mousedownがモーダル内(input内のテキスト選択)で始まってmouseupが背景に来たケースを「背景クリック」と誤検知している

## 修正内容

### 1. 閉じるときに必ずstateをリセット

- モーダルの閉じ処理を担う関数(例: `closeIntake`)を一本化し、以下を必ず実行してから可視stateをfalseにする:
  - url入力 = ""、テキスト入力 = ""、抽出/取得プレビュー結果 = null、エラー文言 = ""、ローディング = false、(URL/テキストのタブ切替があるならタブも初期値へ)
- モーダルを開くトリガー側でも、開く直前に同じ初期化を行う(二重の保険)
- 「トレイに追加」成功後にモーダルを閉じる場合も、この一本化した閉じ処理を通す

### 2. ドラッグでの誤閉じを防ぐ(既存の認証モーダルと同じ手法)

App.jsx内の認証モーダル(`AuthModal`)およびアカウント/レポート系モーダルで既に採用している「mousedownとmouseupが**両方とも背景要素そのもの**のときだけ閉じる」パターンを、この取り込みモーダルにも適用する。参考実装は既存モーダルにある以下の構造をそのまま踏襲すること:

```jsx
const downOnBg = useRef(false);
// オーバーレイ(背景)要素に付与:
onMouseDown={(e) => { downOnBg.current = e.target === e.currentTarget; }}
onMouseUp={(e) => {
  if (downOnBg.current && e.target === e.currentTarget) closeIntake();
  downOnBg.current = false;
}}
```

- 背景に付いている既存の `onClick={close}` は**削除**する(onClickは残さない。mousedown/mouseup方式に一本化)
- モーダル本体(中身のコンテナ)側で `onMouseDown`/`onClick` の伝播が背景に達しないよう、必要なら本体に `onMouseDown={(e)=>e.stopPropagation()}` を付ける
- Escキーで閉じる処理があるなら、それも `closeIntake`(リセット付き)を通す

## 受け入れテスト(PR説明に記載)

1. `npm run build` 成功
2. URLを入力→モーダルを閉じる→再度開くと**入力が空**になっている(テキスト欄・プレビュー・エラーも空)
3. input内のURL文字列を左端から右端までドラッグ選択し、モーダル**背景の上でマウスを離してもモーダルが閉じない**
4. textareaでの同様のドラッグでも閉じない
5. 背景の何もないところを**単純にクリック**したときは従来どおり閉じる
6. 「トレイに追加」成功後に開き直しても入力が残っていない
7. 既存の取り込み機能(URL取得・テキスト抽出・Free上限)に回帰がない
