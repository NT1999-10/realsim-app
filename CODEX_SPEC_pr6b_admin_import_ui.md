# 追補指示書: 管理者CSVインポート画面の追加(PR#6の同一ブランチ)

対象: PR #7(競売タブUI)のブランチ。**同じブランチ上で追加実装**すること。§0の絶対規則は引き続き適用。

## 背景

PR#5で実装した `/api/auction-import` はAPIのみで、ブラウザから操作するUIが無い。運営者(非エンジニア)がBITで確認した物件を日常的に登録できるよう、管理者専用のインポート画面を競売タブ内に追加する。

## 要件

### 1. 表示条件(管理者ゲート)

- ログイン中ユーザーのメールアドレスが管理者である場合**のみ**、競売タブ内に「管理者: 物件データ登録」セクションを表示する
- 管理者判定はサーバー側が正: 軽量なエンドポイント(例: 既存APIに `GET /api/auction-import`(認証付き)で `{isAdmin: true/false}` を返す分岐を追加、または一覧APIのレスポンスに `isAdmin` を含める等、実装しやすい方法でよい)を用意し、クライアントはその結果で表示を切り替える
- **クライアント側に ADMIN_EMAILS の値やメールアドレス一覧を埋め込むことは禁止**(環境変数はサーバー専用)
- 非管理者・未ログイン時はセクション自体を描画しない(LockCard等も不要。存在を見せない)

### 2. 入力UI(2方式を1画面に)

**方式A: フォーム入力(1件ずつ・主要な運用方法)**
- BITの物件明細を見ながら転記できる入力フォーム。項目:
  - 事件番号(case_no・必須)・裁判所名(court)・物件番号(item_no・数値・既定1)
  - 都道府県(pref・47都道府県Select)・市区町村(city)・所在地(address)
  - 種別(type・Select: マンション/戸建て/土地/その他)
  - 売却基準価額(min_price・万円で入力→円に変換して送信)・買受申出保証額(deposit・同様・任意)
  - 入札開始(bid_start)・入札終了(bid_end)・開札日(open_date)— `<input type="date">`
  - 築年(built_year・任意)・建物面積㎡(floor_area・任意)・土地面積㎡(land_area・任意)
  - BITの物件URL(bit_url・必須・https://www.bit.courts.go.jp 配下のみ許可)
- `id` はユーザーに入力させず、クライアントで `court + case_no + item_no` から自動生成(既存APIのid規約に合わせる)
- 「登録する」で既存 `/api/auction-import` にCSV1行分として送信(APIの既存仕様に合わせる。1件でも配列/CSVとして送る)
- 成功時「登録しました」+フォームをクリア。失敗時はAPIのエラーメッセージを日本語で表示

**方式B: CSV貼り付け(まとめて登録・上級者向け)**
- textarea にヘッダー行付きCSVを貼り付け →「まとめて登録」で既存APIへ送信
- textareaの上に、正しいヘッダー行(CSV_COLUMNSの並び)をコピー可能なコードブロックで表示
- 各行の検証エラーはAPIの返却をそのまま行番号付きで表示

### 3. 種別の判定ガイド(フォーム内ヘルプ)

種別Selectの近くに小さなヘルプ(?アイコンまたは常設の注記)で以下を表示:
「区分所有建物(所有権)→マンション ／ 土地+建物(所有権)→戸建て ／ 土地のみ→土地 ／ 借地権付き・共有持分などの特殊な権利→その他」

### 4. 登録済みデータの管理

- 管理者セクション内に「登録済み一覧(直近20件)」を表示: id・所在地・種別・基準価額・入札終了日・active
- 各行に「無効化」ボタン(active=falseにする。物理削除はしない)。無効化APIが無い場合は `/api/auction-import` にactive切替の小さな機能を追加してよい(管理者認証は同一)
- 無効化した物件は一般ユーザーの検索結果から消えることを確認する(auction-list側はactive=trueのみ返す既存仕様の維持)

### 5. 文言

セクション冒頭に運用上の注意を1行: 「BITの公表情報を確認のうえ転記してください。3点セットPDFの保存・転載は行わず、リンクのみ登録します。」

## 受け入れテスト(PR説明に記載)

1. `npm run build` 成功
2. ADMIN_EMAILS のメールでログイン → 競売タブに管理者セクションが表示される
3. 別メール(非管理者)・未ログイン → セクションが一切表示されない。APIへの直接POSTも401/403(既存挙動維持)
4. 方式Aで1件登録 → 一般の検索結果に表示され、NEWバッジ・BITリンク・フォロー・カレンダー連携が機能する
5. 方式Bで2件まとめて登録 → 2件とも表示される。不正な行(必須欠落)はエラーが行番号付きで出る
6. 「無効化」で一覧から消える
7. bit_url にBIT以外のURLを入れると登録が拒否される
